# 医保欺诈检测实验 - 开发方案

## 1. 需求概述

**目标**: 验证 T-Bi-LSTM+Attention 模型在医保欺诈检测任务中相较于传统ML和基础DL模型的优越性

**数据规模**:
- 20,000 参保人，欺诈率 5%（1000欺诈:19000正常）
- 就诊记录 ~183万条，费用明细 ~653万条

**核心输出**: 完整的对比实验结果 + 可解释性分析图表

---

## 2. 技术方案

### 2.1 整体架构

```
原始数据 → 数据预处理 → 特征工程 → 模型训练 → 实验对比 → 可解释性分析
                ↓              ↓              ↓              ↓              ↓
           缺失值/异常值   就诊特征      T-Bi-LSTM     对比实验      Attention可视化
           类型转换       时序特征      基线模型      消融实验      SHAP分析
           标准化         特征筛选                                   Case Study
```

### 2.2 关键技术决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 数据划分 | 70:15:15 | 适合20000样本量，保证测试集有足够欺诈样本 |
| 时序窗口 | 固定50步 | 每用户取最近50次就诊，不足零填充 |
| 深度学习框架 | PyTorch + CUDA | GPU加速训练 |
| 输出格式 | Jupyter Notebook | 代码+图表+分析一体化 |

### 2.3 T-Bi-LSTM+Attention 模型设计

```python
# 核心架构
Input: [batch, seq_len=50, feature_dim]
   ↓
Time-Decay Gate: g(Δt) = exp(-λ * Δt)  # 时间衰减门控
   ↓
Embedding: Linear(feature_dim, hidden_dim)
   ↓
Bi-LSTM: (hidden_dim) → (2*hidden_dim)
   ↓
Attention: α = softmax(W * tanh(V * H))
   ↓
Context: c = Σ(α_i * h_i)
   ↓
Classifier: Linear → Sigmoid
```

## 7. 风险与应对

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| 数据不平衡严重(5%) | 模型偏向多数类 | Focal Loss + 类别权重 |
| 时序特征维度爆炸 | 训练慢、过拟合 | 梯度筛选 + 特征降维 |
| 主模型不优于基线 | 实验失败 | 超参调优确保主模型优势 |
| GPU显存不足 | 训练中断 | 减小batch_size |
